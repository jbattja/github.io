Partner Onboarding and Management APIs

1. Overview​

OpenFabric accelerates the adoption of a Payment Method(BNPLs, Wallets, etc.) offered by our Tenants with their Partners (Merchants, Payment Gateways, etc.) by simplifying the technical integration needed between them.

With OpenFabric, our Tenants can drastically reduce the complexity and time required for their Partners (Merchants and Payment Gateways) to integrate with their Payment Methods.With a simple integration using OpenFabric SDK, a Partner can initiate a transaction, redirect the customer to our Tenant's Payment Method for approval, and get a payment token created for this transaction.

For Partners to integrate and use OpenFabric SDK, Partners need to be onboarded to OpenFabric. The onboarding can be done via our Tenant Servicing Portal or APIs.

Once a Partner is onboarded, OpenFabric generates unique integration credentials for the Partner. We can also configure each Partner's Webhook notification settings and Payment Gateway Settings for token handover.

In this document, let us look at how Tenants can Onboard and Manage Partners via our APIs.

2. Partner Onboarding and Management​

OpenFabric provides the below APIs to create, update, list, and deactivate Partners.

ActionAPI
Create a PartnerPartner Create API
List all PartnersPartner List API
Get Partner DetailsPartner Details API
Update a PartnerPartner Update API
Deactivate a PartnerPartner Deactivate API

3. Credentials Management​

Once a Partner is onboard, OpenFabric will generate unique integration credentials for the Partner. Tenants can use Partner Settings APIs to retrieve and renew credentials (to create a new set of credentials) for a Partner.

ActionAPI
Retrive CredentialsCredentials Retrive API
Renew CredentialsCredentials Renew API

Tenants can integrate into these APIs to display/renew their partners' credentials directly through their servicing portals.

4. Webhook Settings​

Partners can receive webhooks directly from OpenFabric's systems (useful to support transactions that require async response handling). Tenants can use our Partner Settings APIs to manage webhook subscriptions.

ActionAPI
Create a Webhook Subscription for a PartnerSubscription Create API
List all Webhook Subscription for a PartnerSubscriptions List API
Get details of a Webhook SubscriptionSubscription Retrive API
Update Webhook Subscription for a PartnerSubscription Update API
Delete Webhook Subscription for a PartnerSubscription Delete API

Tenants can integrate these APIs into their servicing portal to let their Partners update these settings directly through their servicing portals.

5. Payment Gateway Settings​

Depending on a Partner's needs, OpenFabric can work directly with a Partner's Payment Gateway to generate a PG Token that can be handed over to the Partner in a PCI-compliant manner. OpenFabric, in some scenarios (for Shopping Cart Plugins), can directly charge the virtual card generated for a transaction with the Partner's Payment Gateway.

Tenants can collect and manage the Partners Payment Gateway Credentials using our Partner Settings APIs.

A Partner can have one or more Payment Gateway Credentials against their account.

ActionAPI
Create/Update a Payment Gateway Credentials for a PartnerPayment Gateway Credentials Create API
List all Payment Gateway Credentials configured for a PartnerPayment Gateway Credentials List API
Get details of a configured Payment Gateway CredentialPayment Gateway Credentials Fetch API
Delete a configured Payment Gateway Credential Webhook Subscription for a PartnerPayment Gateway Credentials Delete API
